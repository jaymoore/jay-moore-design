03 / Patterns

Patterns

Cross-component conventions. Each pattern enforces a Phase 0 discipline decision; deviating drifts the identity.

Active state · color only

No layout shift on activation

Active route never adds an underline or border that wasn't already in the inactive state. Color change does the work. Layout stays stable across hover / focus / route change.

Hairline as grid

Hairline borders separate, never decorate

border-line at 1px is the only divider. No section boxes; no enclosed cards for primary content. The hairline is structural — it ties the composition to the page grid.

External link · arrow inherits color

Arrow and label brighten together

External links carry a 16px ArrowUpRight at strokeWidth 1.5. Stroke uses currentColor, so hover brightening the label also brightens the arrow — visually coupled.

Masthead · typeface alternation

Hanken + Spline mono + Hanken

Identity treatments alternate display sans (Hanken) with mono accents (Spline) to signal instrument identity without HUD chrome. Mono is for data, labels, coordinates, and meta — never body.

Light-primary

Light is the designed state; dark is the toggle

The site is designed light-first. Dark is provided via html.dark class with FOUC-prevention script + localStorage + system preference fallback. Tailwind v4's dark: variant is rebound to the class via @custom-variant dark.

Status dot · live signal

Filled circle + soft halo

--color-ok on a 6px circle with a 3px color-mix(in srgb, var(--color-ok) 18%, transparent) halo. The token is reserved for genuine status — never decoration.

04 / Accessibility

Accessibility

WCAG AA is the floor; AAA on body and long-form body. Every interactive element is keyboard reachable with a visible focus ring; reduced motion is honored at the operational layer.

Focus ring

Always visible, accent-colored

Global rule: *:focus-visible { outline: 2px solid var(--color-accent); outline-offset: 2px; border-radius: inherit; }. Tab to the button below to see it.

Selection

Theme-aware foreground

::selection uses --color-accent as background and --color-selection-fg as text (white on light accent, near-black on light-blue dark accent). Highlight the text below to see the contrast.

Selection demonstrates theme-aware foreground — try highlighting this line. Background = accent, text = selection-fg, contrast verified both modes.

Reduced motion

Comprehensive collapse

@media (prefers-reduced-motion: reduce) collapses every animation, transition, and smooth-scroll to 0.01ms. Page reveal, hover transitions, and theme swap all degrade to instant. No state becomes unreachable.

05 / Voice

Voice

Terse, restrained, instrument-coded. Letters do the work; copy never apologizes and never decorates. Jay's own tone, applied consistently across UI surfaces.

Tone

What the system sounds like

  • One idea per paragraph; one paragraph per beat.
  • Active voice. Concrete nouns. No hedging.
  • No marketing tropes ("join the movement," "level up," "unlock").
  • No fragment-manifesto stacks ("Make it fast. Make it beautiful.") — that is Rauno's move; we don't borrow it.
  • Mono labels are quiet (--text-2xs, uppercase, fg-faint); they orient, never shout.

Capitalization

Sentence case body, lowercase tags

  • Body, headings, link labels: sentence case (Design engineer for AI-native workflows).
  • Mono coordinate labels and tags: lowercase (now / prior / open to).
  • Mono section eyebrows: uppercase with tracking (01 / POSITIONING).
  • Brand wordmark: Jay Moore — title case, never all caps.

Abbreviations

Spelled out unless saving meaningful width

  • Expand abbreviations in body and headings (Simple Path Media, not SPM).
  • Mono coordinate labels may abbreviate when row width matters (SPM in now label).
  • Acronyms uppercase (AI, UI, UX, API).
